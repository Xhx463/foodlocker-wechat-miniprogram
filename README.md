# 递煌小黄柜 (Di Huang Little Yellow Locker)

## 项目简介

递煌小黄柜是一个集外卖柜与快递柜功能于一体的智能解决方案，为商家、骑手、快递员和用户提供便捷的取餐和寄件服务。

## 技术架构

- **开发框架**：微信小程序原生开发
- **前端技术**：WXML + WXSS + JavaScript
- **无第三方依赖**：纯原生实现，无npm包或第三方组件库

## 功能模块

### 1. 首页
- 系统状态展示（柜子总数、可用数量、使用率）
- 格口大小分布（小/中/大）
- 快速导航到各功能端

### 2. 用户端
- **取餐服务**：输入取餐码取餐
- **寄件服务**：填写寄件信息，选择包裹大小

### 3. 骑手端
- **存餐服务**：输入商家信息和取餐码，选择柜体温度
- **取件服务**：输入快递单号取件
- **柜子状态**：实时展示柜子使用情况

### 4. 快递小哥端
- **取件服务**：输入快递单号取件
- **派件服务**：填写收件信息，选择格口大小，生成取件码
- **柜子状态**：实时展示柜子使用情况

### 5. 商家管理端
- **系统状态**：详细的系统运行数据
- **柜子管理**：查看所有柜子状态，支持点击查看详情
- **订单管理**：查看最近订单记录
- **操作功能**：刷新状态、导出数据

## 组件使用情况

本项目使用微信小程序原生组件，未引入任何第三方组件库：

| 组件类型 | 具体组件 | 使用场景 |
|---------|---------|----------|
| 基础组件 | view | 布局容器 |
| 基础组件 | text | 文本显示 |
| 基础组件 | button | 操作按钮 |
| 基础组件 | input | 表单输入 |
| 基础组件 | picker | 选择器（快递类型） |
| 逻辑组件 | wx:if | 条件渲染 |
| 逻辑组件 | wx:for | 列表渲染（柜子状态、订单） |
| 事件处理 | bindtap | 点击事件 |
| 事件处理 | bindinput | 输入事件 |

## 特色功能

1. **温度控制**：支持恒温、低温、保温三种温度模式
2. **格口分类**：小格口（外卖）、中格口（一般快递）、大格口（大件快递）
3. **实时状态**：动态展示柜子使用情况和温度状态
4. **响应式设计**：适配不同屏幕尺寸
5. **美观界面**：现代化的UI设计，符合各端用户审美

## 部署说明

1. **克隆项目**
   ```bash
   git clone https://github.com/Xhx463/foodlocker-wechat-miniprogram.git
   ```

2. **导入项目**
   - 打开微信开发者工具
   - 选择「导入项目」
   - 填写项目目录和AppID（或使用测试号）

3. **预览与调试**
   - 在开发者工具中预览
   - 扫码在手机上查看效果
